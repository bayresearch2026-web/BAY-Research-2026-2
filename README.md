# BAY 리서치팀 뉴스트래킹 — 노션 연결 배포 안내

노션 표에 뉴스를 한 줄씩 쌓으면 이 홈페이지에 자동으로 표시됩니다.
코드는 건드릴 필요 없고, 아래 순서대로 클릭·복붙만 하면 됩니다.

## 폴더 구성
- `index.html` — 홈페이지(디자인/화면). 발행일로 주차를 자동 계산합니다.
- `api/posts.js` — 노션에서 데이터를 가져오는 중계 함수
- `package.json` — 배포 설정(안 건드림)

---

## 1. 노션 데이터베이스 구조 (중요)

**뉴스 1건 = 표의 한 줄**이 되도록, 뉴스를 **하나의 데이터베이스**에 모아 쌓습니다.
(주마다 새 페이지/표를 따로 만들면 표 ID가 매주 달라져 연결이 깨집니다.
 대신 이 표 하나를 노션에서 '주차별 뷰'로 묶어 보면 지금처럼 주간정리로 보입니다.)

아래 속성(열)을 두세요. **이름은 영어 그대로** 두는 게 안전합니다(코드가 이 이름으로 읽음).

| 속성 이름 | 타입 | 비고 |
|---|---|---|
| Title | 제목(Title) | 이미 있음 |
| Author | 사람(Person) 또는 선택 | **뉴스 행에 추가** (지금은 주간 페이지에만 있음) |
| Date of Issue | 날짜(Date) | 이미 있음. 이 날짜로 주차 자동 계산 |
| Content Summary | 텍스트 | 이미 있음 → 요약 |
| Insight | 텍스트 | 인사이트 (아래 설명) |
| Source | URL | 이미 있음 |
| Tag | 다중 선택(Multi-select) | 선택사항 (RWA·규제동향·프로젝트…) |
| Status | 상태(Status) | 값이 `홈페이지 게시`인 뉴스만 사이트에 게시. `미완`·미선택은 숨김 |

**인사이트는 두 가지 방식 다 지원합니다:**
- `Insight` 속성 칸에 텍스트로 적으면 → 그대로 사용
- 비워두면 → 그 뉴스 페이지 **본문**(소제목+문단)을 읽어와 인사이트로 사용
→ 지금처럼 본문에 정리해 오셔도 그대로 나옵니다.

**주간 페이지를 예쁘게 유지하는 법 (Linked view):**
데이터는 마스터 DB 하나에 두되, 각 주간 페이지에는 그 DB의 **연결된 보기(linked view)**를 넣고
해당 주로 필터하면 됩니다. 데이터는 한 곳(=한 ID)에 모이고, 보기만 주별로 예쁘게 나뉩니다.
- 주간 페이지에서 `/linked` 입력 → "Linked view of database" → 마스터 DB 선택
- 그 보기에 필터: Date of Issue가 그 주 범위(또는 Week 속성 = 해당 주)
- 연결된 보기에서 행을 추가하면 마스터 DB에 자동으로 들어갑니다.

## 2. 노션 API 토큰 발급
1. https://www.notion.com/my-integrations → **New integration**.
2. 이름(예: BAY 뉴스트래킹) + **데이터베이스가 있는 워크스페이스** 선택 → 생성.
3. **Internal Integration Secret**(토큰) 복사 → 이게 `NOTION_TOKEN`. (비밀번호처럼 취급)

## 3. 데이터베이스를 통합에 연결 (꼭 필요)
1. 그 데이터베이스를 **전체 페이지로** 엽니다.
2. 우측 상단 `⋯` → **Connections(연결)** → 만든 통합 선택.
3. 주소창의 DB ID 복사: `notion.so/`**`32자리 문자열`**`?v=...` → 이게 `NOTION_DB_ID`.

## 4. Vercel에 배포 (무료)
1. **GitHub** 가입 → New repository → `Add file` → `Upload files`
   → 이 폴더 파일들을 **구조 그대로** 업로드 → Commit.
2. **Vercel** 가입(GitHub로 로그인) → Add New → Project → 그 repo Import.
3. **Environment Variables**에 두 개 추가:
   - `NOTION_TOKEN` = (2번 토큰)
   - `NOTION_DB_ID` = (3번 DB ID)
4. **Deploy** → 공개 URL 완성.

## 5. 운영 & 인수인계
- 노션 표에 한 줄 추가 = 뉴스 1건. 수정도 노션에서 그 줄 열어 고치면 됩니다.
- 팀원에게는 노션 표 **편집 권한**만 부여.
- 인수인계: **노션 공용 계정 + Vercel(또는 GitHub) 로그인 정보**만 넘기면 끝.

## 디자인 수정
- `index.html`의 HTML/CSS만 바꾸면 됩니다(데이터 연결과 무관, 언제든 가능).
- 수정 후 GitHub에 다시 올리면 Vercel이 자동 재배포합니다.

## 문제가 생기면
- 사이트에 샘플만 보이면: 환경변수(2·3번)·통합 연결(3번)·열 이름을 확인.
- `배포주소/api/posts` 를 직접 열면 오류 메시지가 보여 원인 파악에 도움이 됩니다.
